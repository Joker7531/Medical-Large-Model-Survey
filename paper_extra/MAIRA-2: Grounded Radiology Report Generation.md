## 二、研究概述

**2.1 研究背景与动机**

> 放射科报告需求持续增长，而放射科医生供给不足，导致职业倦怠与医疗差距扩大。现有AI报告生成模型存在两大核心缺陷：生成内容难以被医生快速核验（缺乏可溯源性），以及缺乏对先验影像、临床适应症等上下文的综合利用。MAIRA-2 旨在同时解决这两个问题。

**2.2 核心贡献**

- **贡献1：** 提出"有依据的放射报告生成"（Grounded Radiology Report Generation）新任务，要求模型在生成每条发现描述的同时，输出对应的空间定位边界框，提升报告可验证性。
- **贡献2：** 构建 MAIRA-2 多模态模型，支持同时输入当前正位/侧位片、历史正位片、历史报告及临床文本（适应症、技术说明、对比说明），全面模拟放射科医生的实际工作流。
- **贡献3：** 提出 **RadFact** 评估框架，基于 LLM（Llama3-70B）进行句子级逻辑蕴含推断，突破传统 BLEU/ROUGE 和固定病种分类指标的局限，支持逻辑精确率/召回率及空间定位精确率/召回率的多维评估。

**2.3 临床应用场景**

- **科室：** 放射科
- **疾病类型：** 胸部X光相关疾病（肺炎、胸腔积液、气胸、肺结节等）
- **应用阶段：** 报告生成（草稿辅助）+ 诊断辅助

## 三、技术方法

**3.1 模型架构**

| 字段 | 内容 |
|------|------|
| **架构类型** | VLM（视觉语言模型），基于 LLaVA 架构 |
| **视觉编码器** | Rad-DINO-MAIRA-2（ViT-B，87M 参数），在 140 万张医学影像上重新训练 |
| **语言模型** | Vicuna 7B v1.5 |
| **适配器** | 四层 MLP，连接视觉编码器与语言模型 |
| **模型总参数量** | ~7B+（视觉编码器 87M + 语言模型 7B） |

**3.2 训练策略**

- **预训练方式：** 迁移学习 + 指令微调（SFT），多任务联合训练，共 3 个 epoch
- **优化器：** AdamW
- **上下文扩展：** 线性 RoPE 缩放因子 1.5，以支持三张图像 + 长文本输入
- **提示工程：** ✅ 是，使用结构化 Prompt 交织图像 token 与文本字段
- **多阶段训练：** 否（单阶段多任务）

**3.3 多模态信息**

| 字段 | 内容 |
|------|------|
| **涉及模态** | 医学图像（CXR）+ 文本（报告、适应症、技术说明） |
| **融合方式** | 投影层对齐（MLP Adapter）+ 序列拼接 |
| **对齐策略** | 投影层将视觉 token 映射至语言模型空间，图像 token 与文本 token 交织输入 |
| **图像分辨率** | 518×518，切分为 14×14 patches，每张图像产生 1,369 个视觉 token |
| **空间定位编码** | 100×100 网格坐标 token（横纵坐标使用不相交 token 集合编码） |

## 五、实验与结果

**5.1 评估指标**

- 传统指标：BLEU-4、ROUGE
- 临床指标：CheXbert Macro F1-14、RadGraph-F1
- 新提出指标（RadFact）：
  - 逻辑精确率 / 逻辑召回率
  - 定位精确率 / 定位召回率
  - 空间精确率 / 空间召回率
- 空间蕴含判定标准：生成框中 ≥50% 像素落在参考框联合区域内

**5.2 对比基线**

| 模型 | BLEU-4 | RadGraph-F1 |
|------|--------|-------------|
| **MAIRA-2** | **23.1** | **34.6** |
| MedVersa | 17.8 | 28.0 |
| MAIRA-1 | 14.2 | — |

**5.3 核心结果**

> MAIRA-2 在 MIMIC-CXR 基准上达到 SOTA，BLEU-4 为 23.1，RadGraph-F1 为 34.6，均显著超越前代模型和对比方法。在外部数据集 IU-Xray 上，RadFact 逻辑精确率 71.4%、召回率 67.6%，展现出良好的泛化能力。在有依据报告生成任务（GR-Bench）上，逻辑精确率 74.1%，定位召回率高达 90.6%，表明模型能可靠覆盖真实发现的空间位置。专家评审显示 91% 的生成句子可直接接受，70% 的报告仅需少于两处修改。

**5.4 消融实验**

- ✅ **时序上下文（历史片 + 对比说明）：** 去除后模型出现"幻觉性时序比较"（无历史片却输出"unchanged"），且 CheXbert F1 和 RadFact 召回率显著下降；对比说明字段是触发时序语言的关键，历史图像/报告保证时序描述的准确性。
- ✅ **多视角上下文（侧位片 + 技术说明）：** 去除后侧位特异性发现准确率下降；胸腔积液检测 F1 从 71.4 降至 64.7，验证侧位视图的临床价值。

## 六、局限性与不足

**作者自述：**
- **召回缺口：** 模型仍存在漏诊（遗漏发现），是临床部署前最需改进的方向
- **内部一致性：** RadFact 逐句评估，无法检测报告内部的自相矛盾
- **数据约束：** 有依据报告数据集（如 GR-Bench）缺乏历史片等完整上下文输入，限制了对上下文如何影响定位的深入研究
- **临床权重缺失：** RadFact 对所有逻辑错误一视同仁，未区分临床严重程度

**整理者补充：**
- 私有数据集（USMix、GR-1、GR-Bench）占训练数据约 51%，外部研究者无法完全复现，限制了方法的可重复性
- 模型目前仅支持胸部X光，向其他影像模态（CT、MRI）的扩展路径尚未探索
- 专家评审样本量仅 20 例，统计效力有限，难以代表真实临床场景的全部多样性

---

## 七、挑战与未来方向

- **提升召回率：** 减少发现遗漏，是模型走向临床辅助的核心瓶颈
- **临床显著性加权评估：** 开发能区分错误临床严重程度的评估指标，使评估更贴近真实医疗价值
- **扩展上下文完整性：** 构建包含完整输入（历史片、历史报告、临床适应症）的有依据报告数据集
- **跨模态与跨机构泛化：** 探索模型在不同医院、不同设备、不同影像模态下的鲁棒性
