## 二、研究概述

**2.1 研究背景与动机**

深度学习在电子健康记录（EHR）驱动的疾病预测中表现出色，但通常需要大量标注训练数据，而真实临床场景中标注数据往往稀缺。BERT 在 NLP 领域通过大规模预训练显著提升了小数据任务的性能，但此前 BERT 类模型仅被应用于临床文本，尚未被系统性地适配到**结构化 EHR 数据**（如 ICD 诊断码序列）。Med-BERT 正是为填补这一空白而提出。

**2.2 核心贡献**

- **贡献1**：首次将 BERT 框架系统性地适配至结构化 EHR 领域，设计了专用的多层嵌入（代码嵌入 + 序列化嵌入 + 就诊嵌入），解决了 EHR 与自然语言在数据结构上的本质差异。
- **贡献2**：提出了领域特定的预训练任务——**住院时长延长预测（Prolonged LOS）**，该任务无需人工标注、覆盖跨就诊上下文，有效增强了模型对 EHR 序列语义的捕捉能力。
- **贡献3**：在两个疾病预测任务（糖尿病心衰预测 DHF、胰腺癌预测 PaCa）的三个队列上验证了 Med-BERT 的跨数据库泛化能力，并证明其在**极小样本场景**下可将 AUC 提升超过 20%，等效于训练集扩大 10 倍的效果。

**2.3 临床应用场景**

- **科室**：全科 / 内分泌科 / 肿瘤科 / 心内科
- **疾病类型**：糖尿病合并心力衰竭（DHF）、胰腺癌（PaCa）
- **应用阶段**：疾病预测（早期预警 / 诊断辅助）

## 三、技术方法

**3.1 模型架构**

- **架构类型**：Transformer（双向编码器，BERT 风格）
- **具体模型**：Med-BERT（基于原版 BERT 改造，适配结构化 EHR）
- **模型参数量**：约 **1700 万参数**（L=6层，H=192隐层维度，A=6注意力头，FFN=64）

**3.2 训练策略**

- **预训练方式**：自监督预训练（Masked LM + Prolonged LOS 预测），在 2849 万患者数据上训练超过 4500 万步
- **下游任务**：迁移学习 + 微调（Fine-tuning），预训练权重加载后与任务头联合优化
- **是否使用提示工程**：否
- **是否涉及多阶段训练**：是 —— 第一阶段大规模自监督预训练（Cerner，2000万患者），第二阶段任务特定微调（DHF/PaCa 队列）

**3.3 多模态信息**

- **不适用**（当前版本仅使用结构化诊断码，未融合影像、文本等其他模态）
- 作者明确指出将在未来工作中加入药物、检验、手术等其他编码类型

**4.2 数据处理**

- **预处理方式**：按时间顺序排列就诊记录；依据"入院时诊断标志 → 就诊期间记录 → Cerner优先级"三级规则对就诊内诊断码排序；剔除诊断码少于3个及日期错误的患者
- **数据增强策略**：无（依赖大规模预训练数据本身的覆盖广度）
- **隐私保护措施**：去标识化（符合 HIPAA 合规要求，IRB 审批通过，协议号 HSC-SBMI-13-0549）

## 五、实验与结果

**5.1 评估指标**

- 主要指标：**AUC**（AUROC）
- 辅助指标：准确率（Accuracy）、F1、精确率、召回率（见补充材料 Table 1-2）

**5.2 对比基线**

| 类别 | 模型 |
|------|------|
| 经典深度学习 | GRU、Bi-GRU、RETAIN |
| 静态嵌入增强 | GRU/Bi-GRU/RETAIN + t-W2V（时间敏感 Word2Vec） |
| 传统机器学习 | L2正则化逻辑回归（L2LR）、随机森林（RF） |
| EHR BERT 类对比 | BEHRT、G-BERT（定性比较） |
| 消融对照 | Untrained Med-BERT（随机初始化，不预训练） |

**5.3 核心结果**

在三个评估队列上，Med-BERT 均带来显著提升：

- **DHF-Cerner**：Bi-GRU + Med-BERT 达到 AUC **85.39**，较 Bi-GRU 基线（82.82）提升 **+2.57%**
- **PaCa-Cerner**：AUC 最高达 **82.23**，较基线 GRU（78.26）提升 **+6.14%**（最大提升幅度）
- **PaCa-Truven（跨库泛化）**：AUC 最高 **80.57**，较基线提升 **+3.78%**，验证了跨数据库迁移能力
- **小样本场景**：仅用 **300 个样本**时，Bi-GRU + Med-BERT 可达 AUC 0.75；GRU + Med-BERT 用 1000 样本即可达到 GRU 单独使用 10,000 样本的水平

**5.4 消融实验**

有消融实验。对比了**预训练 Med-BERT** vs **未预训练 Med-BERT（随机初始化）**：
- 未预训练版本在 PaCa 任务上甚至不如逻辑回归，说明**预训练阶段是性能提升的核心来源**，而非单纯依赖 Transformer 结构本身
- 同时验证了 Prolonged LOS 预训练任务的有效性（相比仅使用 Masked LM）

## 六、局限性与不足

**作者自述：**
- 当前版本仅使用 ICD 格式的**诊断信息**，未纳入药物、检验、手术等其他 EHR 数据类型
- 未建模就诊之间的**时间间隔**，可能导致部分时序信息丢失
- 就诊内诊断码的排序基于账单优先级，**可靠性有限**，未充分探索更优的排序策略

**整理者补充：**
- 预训练数据（Cerner）和部分评估数据来自同一来源，跨机构泛化能力仅通过 Truven 一个外部数据集验证，**多中心泛化性**尚待更充分评估
- 模型架构参数较小（H=192），与后续大规模语言模型相比表达能力有限，在更复杂任务上的上限不明确
- 仅处理**单模态结构化数据**，与临床实际中多模态信息融合的需求存在差距

## 七、挑战与未来方向

- 将药物、实验室检测、手术操作等**多类型 EHR 编码**纳入 Med-BERT 输入，构建更全面的预训练模型
- 探索 **ULMFiT、ELMo、GPT** 等其他预训练框架在结构化 EHR 上的适配效果
- 设计更多**领域特定预训练任务**，并测试疾病预测以外的下游任务（如再入院预测、治疗推荐）
- 开发更精细的**任务特定可视化与可解释性**工具，以增强临床信任度

**8.3 整理者评价**

Med-BERT 是将 BERT 预训练范式引入**结构化 EHR** 领域的奠基性工作，方法设计严谨，尤其是 Prolonged LOS 预训练任务的设计颇具创意。其在小样本场景下的显著增益具有重要临床价值，但受限于单模态输入和较小的模型规模，在当前大模型背景下更多作为**方法论先驱**而非最优实践。
