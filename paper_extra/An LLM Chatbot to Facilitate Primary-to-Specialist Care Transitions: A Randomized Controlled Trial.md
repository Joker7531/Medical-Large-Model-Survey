## 二、研究概述

### 2.1 研究背景与动机

中国医疗系统面临严峻的门诊压力：2023年医院就诊量达42.6亿次次，而初级医疗资源严重不足，大量患者绕过基层直接涌入三甲医院专科门诊，导致专科医生在缺乏转诊信息的情况下接诊，造成诊断效率低下、医生职业倦怠和患者体验差。现有LLM应用多局限于医生辅助工具，缺乏在资源有限、高流量真实临床环境中直接面向患者的实证验证。

### 2.2 核心贡献

- **贡献1：** 开发了PreA（Pre-Assessment）——一款基于GPT-4o mini、通过多利益相关方参与式共同设计（co-design）构建的LLM聊天机器人，可自主完成病史采集、初步诊断建议和检查推荐，并生成结构化转诊报告。
- **贡献2：** 在中国两家三甲医院开展了迄今规模最大的患者直面LLM临床RCT（n=2,069），证明PreA在减少问诊时长（-28.7%）、提升医生感知的诊疗协调度（+113.1%）和患者沟通体验（+16.0%）三个主要终点上均达到显著改善。
- **贡献3：** 通过消融实验证明，参与式共同设计策略显著优于在本地对话数据上微调同一模型的策略，为资源有限环境中公平部署医疗AI提供了方法论依据。

### 2.3 临床应用场景

| 维度 | 内容 |
|------|------|
| **科室** | 跨24个医学专科（内科、外科、内外科混合、儿科） |
| **疾病类型** | 不限特定疾病，面向通科门诊转诊场景 |
| **应用阶段** | 初级医疗→专科转诊过渡环节（病史采集 + 诊断辅助 + 报告生成） |

## 三、技术方法

### 3.1 模型架构

| 维度 | 内容 |
|------|------|
| **架构类型** | LLM（大语言模型） |
| **具体模型** | OpenAI GPT-4o mini（ChatGPT-4.0 mini） |
| **模型参数量** | 未公开披露 |
| **系统组成** | 患者端聊天机器人（病史采集） + 医生端临床界面（结构化转诊报告生成） |

### 3.2 训练策略

| 维度 | 内容 |
|------|------|
| **预训练方式** | 基于GPT-4o mini，采用提示工程 + Agent技术进行共同设计对齐，**未**对基础模型进行微调（co-design版本） |
| **对照组策略** | 数据微调版本（data-tuned）：在515个本地初级医疗对话（199,145中文字）上对同一基础模型进行fine-tuning |
| **提示工程** | ✅ 是，采用两阶段临床推理逻辑（询问阶段 + 结论阶段），结合对抗性测试迭代优化 |
| **多阶段训练** | ✅ 两轮共同精炼：第一轮为真实利益相关方对抗测试；第二轮为虚拟患者仿真优化 |

### 3.3 多模态信息

| 维度 | 内容 |
|------|------|
| **涉及模态** | 文本（主要）+ 语音（患者可通过语音或文字输入） |
| **融合方式** | 不适用（单模态文本推理为主） |
| **对齐策略** | 不适用 |

## 四、数据信息

### 4.1 数据集

| 数据集名称 | 数据类型 | 样本规模 | 公开/私有 | 来源机构 |
|-----------|---------|---------|-----------|---------|
| RCT临床数据 | 结构化问卷 + 电子病历 + 平台日志 | 2,069名患者 | 私有（可申请） | 桂林医科大学附属医院 + 甘肃医学院附属医院 |
| 本地初级医疗对话语料 | 去标识化音频转录 | 515个场景（199,145中文字） | 私有 | 中国11个省份农村诊所及社区卫生中心 |
| 虚拟患者档案 | LLM合成病例 | 600个（300用于训练，300用于评估） | 私有 | 研究团队合成，5位执业医师验证 |

### 4.2 数据处理

- **预处理方式：** 音频转录后人工校准（去除噪声、非临床话语、模糊表达、隐性体检引用）；电子病历提取五大临床域（病史、体检、诊断、检查、治疗）
- **数据增强策略：** 虚拟患者仿真（50%需多学科会诊，模拟复杂病例）
- **隐私保护措施：** 全数据去标识化处理，符合HIPAA相关监管标准；原始对话数据不公开

## 五、实验与结果

### 5.1 评估指标

| 类别 | 指标 |
|------|------|
| **效率** | 问诊时长（分钟）；每班次接诊患者数；患者等待时间 |
| **医生体验** | 诊疗协调度（5分Likert）；临床决策有用性；工作量减轻感知；患者-医生沟通促进感知 |
| **患者体验** | 沟通便利性、医生关注度、人际关怀、患者满意度、未来接受意愿（均5分Likert） |
| **报告质量** | 完整性、适当性、临床相关性（5分Likert，双盲专家评分）；与医生病历的一致率 |
| **临床决策分析** | F1分类分数（检测自动化偏差）；Mann-Whitney U检验（五大临床域计数比较） |
| **模型对比** | 同上质量评分，co-design vs. data-tuned |

### 5.2 对比基线

| 对比对象 | 说明 |
|---------|------|
| **No-PreA组** | 标准诊疗流程（仅提供患者性别和年龄） |
| **PreA-human组** | 有工作人员辅助使用PreA（验证自主运行能力） |
| **Data-tuned模型** | 同一GPT-4o mini基础模型在本地对话数据上微调后的版本 |
| **非参与医生（匹配对照）** | 用于评估PreA对医生工作量的系统级影响 |

### 5.3 核心结果

PreA-only组相比No-PreA组，问诊时长显著缩短28.7%（3.14 vs 4.41 min，P<0.001）；医生感知的诊疗协调度提升113.1%（3.69 vs 1.73，P<0.001）；患者报告的沟通便利性提升16.0%（3.99 vs 3.44，P<0.001）。参与PreA的医生每班次多接诊15.3%的患者，且等待时间未显著延长。PreA生成的转诊报告在病史、诊断、检查建议上与医生病历的一致率达65.8%–70.7%，且质量评分显著高于医生手写病历。

### 5.4 消融实验

✅ **有消融实验（预试验对比研究）：** 对比co-design版PreA与data-tuned版PreA（同一基础模型在本地对话上微调）在300个虚拟患者档案上的表现。结果显示co-design版在病史采集（4.56 vs 3.86）、诊断（4.67 vs 2.47）、检查建议（4.23 vs 2.21）三个域均显著优于data-tuned版（P<0.001）。Data-tuned版复制了真实初级医疗中的系统性缺陷，包括遗漏指南推荐的病史要素、30%的病例未提供诊断、39.3%未建议必要检查。

## 六、局限性与不足

### 作者自述

- **情境依赖性：** 研究在高流量、资源有限的中国三甲医院开展，效率提升效果可能不能直接推广至其他医疗体系（如低流量或高资源环境）。
- **单盲设计偏差：** 患者知晓自身分组，存在表现偏差风险（尽管客观指标与主观指标结果一致，一定程度上缓解了这一担忧）。
- **Co-design的局限：** 共同设计方法仍受限于资源有限环境中的数据质量，未来需与高质量初级医疗对话数据集进行比较研究。

### 整理者补充

- **LLM版本锁定风险：** 系统基于GPT-4o mini，随OpenAI模型更新，PreA行为可能发生漂移，长期稳定性有待验证。
- **安全性与幻觉问题未深入讨论：** 论文未系统报告PreA生成错误诊断或有害建议的频率及处理机制，临床安全性评估不够充分。
- **推广成本未量化：** 共同设计流程涉及跨11省、多轮利益相关方工作坊，其实际推广成本和可复制性未作详细分析。

---

## 七、挑战与未来方向

- **家庭场景延伸：** 将PreA从医院门诊扩展至居家使用，需解决AI健康素养、网络连接限制和跨机构数据共享等系统性障碍。
- **多样化医疗系统验证：** 需在不同医疗体系（低流量、高资源、非中国环境）中开展更大规模多中心试验，以验证效果的可泛化性。
- **临床文档辅助：** PreA报告质量显著高于医生手写病历，未来可探索将其作为临床文档辅助工具，缓解高流量环境中的记录缺失问题。
- **Co-design vs. 高质量数据集：** 需开展比较研究，评估参与式共同设计与新兴高质量初级医疗对话数据集的相对优劣。
