## 二、研究概述

**2.1 研究背景与动机**

> 现有医疗AI模型大多是单任务系统，缺乏语言表达与交互能力，与真实临床工作流存在明显落差。现有医疗问答基准评估方式单一（仅依赖多选题准确率或BLEU等自动指标），无法全面衡量LLM在安全关键医疗场景中的表现。因此，亟需构建更全面的评估框架，并探索LLM在医疗领域的对齐方法。

**2.2 核心贡献**

- **贡献1：** 构建 MultiMedQA 综合基准，整合6个已有医疗问答数据集，并新增 HealthSearchQA（3,173条消费者常见健康搜索问题）
- **贡献2：** 提出多维度人工评估框架，涵盖事实性、科学共识一致性、推理能力、潜在危害与偏见等轴向，超越传统自动化指标
- **贡献3：** 提出**指令提示微调（Instruction Prompt Tuning）**技术，构建 Med-PaLM 模型，在消费者医疗问答的多项人工评估指标上接近临床医生水平

**2.3 临床应用场景**

- **科室：** 全科 / 初级保健 / 多科室通用
- **疾病类型：** 通用型，涵盖内科、儿科、外科、药物咨询等
- **应用阶段：** 知识检索、临床决策支持、患者健康咨询、医疗问答

## 三、技术方法

**3.1 模型架构**

- **架构类型：** LLM（大型语言模型）
- **具体模型：** PaLM（540B）、Flan-PaLM（540B）、Med-PaLM（540B，指令提示微调版）
- **模型参数量：** 5400亿参数（主实验模型）；另有8B、62B变体用于消融

**3.2 训练策略**

- **预训练方式：** PaLM 从头预训练（780B tokens）；Flan-PaLM 为指令微调；Med-PaLM 为指令提示微调（Instruction Prompt Tuning）
- **是否使用提示工程：** ✅ 是，包括 Few-shot、Chain-of-Thought（CoT）、Self-Consistency 三种策略
- **是否涉及多阶段训练：** ✅ 是
  - 阶段1：PaLM 大规模预训练
  - 阶段2：Flan-PaLM 多任务指令微调
  - 阶段3：Med-PaLM 基于少量临床示例的指令提示微调（仅更新软提示参数，主模型冻结）

**3.3 多模态信息**

- 不适用（本研究仅涉及文本模态）

## 五、实验与结果

**5.1 评估指标**

- **自动化指标：** 多选题准确率（Accuracy）
- **人工评估指标（临床医生）：** 科学共识一致性、错误/不当内容、遗漏内容、潜在危害程度与可能性、偏见
- **人工评估指标（非专业用户）：** 答案相关性（是否回应问题意图）、帮助性
- **不确定性评估：** 选择性预测（Selective Prediction）准确率 vs. 拒答比例曲线

**5.2 对比基线**

PubMedBERT（100M）、BioLinkBERT（340M）、DRAGON（360M）、GPT-Neo（2.7B）、PubMedGPT（2.7B）、Galactica（120B）、BioGPT（355M）、PaLM（各尺寸）

**5.3 核心结果**

- Flan-PaLM 540B 在 MedQA（USMLE）上达到 **67.6%** 准确率，超越此前最优（PubMedGPT 50.3%）**逾17%**
- MedMCQA 达 57.6%，PubMedQA 达 79.0%，MMLU 专业医学子集达 83.8%，均为当时最优
- 人工评估中，Med-PaLM 科学共识一致率达 **92.6%**，接近临床医生（92.9%），远超 Flan-PaLM（61.9%）
- Med-PaLM 潜在危害答案比例降至 **5.9%**，与临床医生（5.7%）相当，Flan-PaLM 为 29.7%

**5.4 消融实验**

| 消融维度 | 主要发现 |
|---------|---------|
| 模型规模（8B → 540B） | 准确率提升约 2 倍，MedQA 提升超 24% |
| 指令微调（PaLM → Flan-PaLM） | 所有尺寸均有提升，PubMedQA 提升超 30% |
| CoT 提示 | 多选题上未见显著提升，但为 Self-Consistency 提供基础 |
| Self-Consistency | MedQA 提升超 7%，PubMedQA 略有下降 |
| 指令提示微调（→ Med-PaLM） | 长文本生成质量大幅提升，安全性和偏见指标显著改善 |

## 六、局限性与不足

**作者自述：**
- MultiMedQA 仅覆盖英语数据集，缺乏多语言支持
- 人工评估规模有限（每题仅1名评估者），评估者池较小且多样性不足
- 偏见评估框架较为初步，未能全面覆盖健康公平性与结构性偏见问题

**整理者补充：**
- Med-PaLM 未开源模型权重，可复现性受限
- 指令提示微调仅使用65个示例，泛化能力有待更大规模验证
- 评估场景与真实临床工作流仍有差距（如多轮对话、电子病历整合等场景未涉及）

## 七、挑战与未来方向

- **知识时效性：** 模型反映训练时的医学共识，需探索持续学习与动态知识检索机制
- **不确定性表达：** LLM 需具备有效传达自身不确定性的能力，避免过度自信输出
- **多语言扩展：** 扩展 MultiMedQA 至非英语语言，服务全球患者
- **公平性与健康公平：** 建立系统性偏见检测与缓解框架，防止加剧健康差距
- **真实临床部署：** 从实验室评估走向前瞻性临床试验，需配套监管框架与安全护栏
