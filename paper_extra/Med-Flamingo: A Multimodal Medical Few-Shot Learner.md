## 二、研究概述

### 2.1 研究背景与动机

医学本质上是多模态领域，临床诊断需要综合图像、文本、检验数据等多种信息。现有医疗视觉语言模型（VLM）通常需要在大量下游数据上微调，但医疗场景中标注数据极为稀缺。因此，**能够从少量示例中实时学习**的多模态医疗基础模型具有重要的临床价值。

### 2.2 核心贡献

- **贡献1：** 提出首个面向医疗领域的多模态少样本学习器 Med-Flamingo，支持图文交错的上下文学习（in-context learning），无需微调即可适应新任务
- **贡献2：** 构建了基于 4,721 本医学教科书的大规模交错图文数据集（MTB），共约 80 万张图像、5.84 亿 tokens，为医疗多模态预训练提供高质量数据基础
- **贡献3：** 创建了 Visual USMLE 评测集（618 道跨专科开放式问题），并设计了首个生成式医疗 VQA 人工评估流程（含专科医生盲评 App）

### 2.3 临床应用场景

- **科室：** 全科（覆盖放射科、病理科、心内科、皮肤科、神经科等 49 个专科）
- **疾病类型：** 不限特定疾病，涵盖 USMLE 风格的跨专科临床问题
- **应用阶段：** 诊断辅助 / 鉴别诊断 / 诊断推理（rationale generation）

## 三、技术方法

### 3.1 模型架构

| 项目 | 内容 |
|------|------|
| **架构类型** | VLM（视觉语言模型） |
| **具体模型** | 基于 OpenFlamingo-9B；语言主干为 LLaMA-7B（冻结）；视觉编码器为 CLIP ViT/L-14（冻结） |
| **可训练参数量** | 1.3B（门控交叉注意力层 + Perceiver Resampler层） |
| **总参数量** | 约 8.3B |

### 3.2 训练策略

- **预训练方式：** 在 OpenFlamingo 检查点基础上进行**领域继续预训练**（continued pre-training），非从头训练
- **是否使用 Prompt Engineering：** ✅ 是，推理阶段采用多模态少样本提示（few-shot prompting）
- **多阶段训练：** 否，单阶段联合训练（paired + interleaved 数据联合目标）
- **训练细节：** 8× 80GB A100，DeepSpeed ZeRO Stage 2，8-bit AdamW，训练 2700 步（约 6.75 天），总 batch size 400

### 3.3 多模态信息

| 项目 | 内容 |
|------|------|
| **涉及模态** | 文本 + 图像 |
| **融合方式** | 跨模态注意力（门控交叉注意力，Gated Cross-Attention） |
| **对齐策略** | Perceiver Resampler 将视觉特征压缩后投影至语言空间；图文交错建模 |

## 四、数据信息

### 4.1 数据集

| 数据集名称 | 数据类型 | 样本规模 | 公开 / 私有 | 来源机构 |
|-----------|---------|---------|------------|---------|
| **MTB**（Medical TextBooks） | 交错图文（教科书） | ~80万图像，5.84亿tokens | 私有构建 | Stanford（自建） |
| **PMC-OA** | 配对图文（论文图注） | 训练130万对，验证16万对 | 公开 | PubMedCentral |
| **VQA-RAD** | 放射科 VQA | 自定义切分（90/10） | 公开 | 文献来源 |
| **PathVQA** | 病理科 VQA | ~6700张测试图（去重后） | 公开 | 文献来源 |
| **Visual USMLE** | 跨专科开放式VQA | 618题 | 私有（Amboss平台授权） | Stanford 自建 |

### 4.2 数据处理

- **预处理：** PDF→HTML 转换，保留 `<image>` 标记；去重 + 内容过滤；按图像数量（≤10）和最大长度切段
- **数据增强：** 未提及
- **隐私保护：** 数据均来自公开出版物和授权教材，未涉及患者数据，无需去标识化
- **去泄漏处理：** 使用 Google ViT 嵌入 + FAISS 近邻检索，识别并剔除预训练集与评测集的重复图像（PathVQA 中剔除 194 张）

## 五、实验与结果

### 5.1 评估指标

- **临床评估分（Clinical Eval Score）**：由3位医生（含1位认证放射科医生）盲评，0–10分，**主要指标**
- **BERT-sim**：生成答案与标准答案的 BERTScore F1
- **Exact-match**：精确匹配率（忽略标点）

### 5.2 对比基线

| 基线模型 | 说明 |
|---------|------|
| MedVINT zero-shot | 基于 LLaMA + PMC-CLIP 的指令微调医疗 VLM |
| MedVINT fine-tuned | 在各数据集训练集上微调后的 MedVINT |
| OpenFlamingo zero-shot | 通用域 VLM，Med-Flamingo 的基础模型 |
| OpenFlamingo few-shot | 通用域 VLM 少样本版本 |
| Med-Flamingo zero-shot | 本文模型零样本版本 |

### 5.3 核心结果

**Med-Flamingo few-shot 在三个数据集上平均排名最优（平均排名 1.67，次优为 OpenFlamingo zero-shot 的 2.33）。**

| 数据集 | 最优模型 | 临床评分 | 提升幅度 |
|--------|---------|---------|---------|
| VQA-RAD | Med-Flamingo few-shot | **5.61** | +20%（vs. 最佳基线 4.69） |
| PathVQA | OpenFlamingo few-shot | 2.16 | Med-Flamingo 表现相对弱 |
| Visual USMLE | Med-Flamingo few-shot | **4.33** | 略优于 OpenFlamingo zero-shot（4.31） |

### 5.4 消融实验

论文未设置严格消融实验，但通过 zero-shot vs. few-shot 对比，验证了**少样本提示策略的有效性**（VQA-RAD 上 few-shot 显著优于 zero-shot）；同时通过与 OpenFlamingo 对比，验证了**医疗领域继续预训练的价值**。

## 六、局限性与不足

**作者自述：**
- Med-Flamingo 仅为概念验证（proof-of-concept），**不适合也不安全用于临床实践**
- 所有模型（包括基线）均存在**幻觉（hallucination）**问题，可能生成低质量输出
- 缺乏指令微调（instruction tuning）或偏好对齐（RLHF），偶尔输出质量不稳定

**整理者补充：**
- PathVQA 上表现较弱，说明模型对**细粒度病理图像**的泛化能力有限，训练数据中病理比例偏低
- Visual USMLE 的 few-shot 提示因问题过长需用 GPT-4 压缩，引入了**外部模型依赖**，影响端到端可复现性
- 评估规模较小（每数据集仅 50 题人工评估），统计显著性有待加强

---

## 七、挑战与未来方向

- **数据扩展：** 引入高分辨率医学图像数据集、3D 体积数据（CT/MRI）及医疗视频，突破当前"只懂书本"的局限
- **临床数据学习：** 从真实患者数据中学习，而非仅依赖出版物，以提升下游临床任务性能
- **对齐与可解释性：** 结合指令微调、RLHF 及多模态检索增强（RAG），提升生成质量与可信度
- **规模化提升：** 更大的模型规模与更丰富的数据多样性预期带来显著性能提升
