## 二、研究概述

**2.1 研究背景与动机**

> 现有的基于 BERT 的 EHR 表示学习方法（如 BEHRT、MedBert）将患者病历视为文本序列，忽略了临床事件之间的**时间间隔信息**，导致模型无法充分利用 EHR 数据的时序特性。此外，BERT 原始的第二学习目标（NSP）在结构化 EHR 场景下并不适用，存在明显的领域适配缺口。该研究旨在通过融合多种时序编码方式，提升 BERT 在疾病预测任务中的性能。

**2.2 核心贡献**

- **贡献1：** 提出人工时间标记（Artificial Time Tokens, ATT）机制，将相邻就诊之间的时间间隔以离散 token 形式插入患者序列，使 BERT 能感知时序间隔。
- **贡献2：** 将概念嵌入（concept embedding）、时间嵌入（time embedding）和年龄嵌入（age embedding）拼接后经全连接层融合，形成"时序概念嵌入"，同时编码绝对时间与相对年龄信息。
- **贡献3：** 设计了新的第二学习目标——就诊类型预测（Visit Type Prediction, VTP），替代 NSP，利用 EHR 中就诊类型与医疗概念的关联关系增强表示学习。

**2.3 临床应用场景**

- **科室：** 内科 / 心内科 / 全科（多科室通用）
- **疾病类型：** 心力衰竭（HF）、2型糖尿病（T2DM）、全因死亡、住院风险
- **应用阶段：** 疾病预测（预后预测）、风险分层、再入院预测

## 三、技术方法

**3.1 模型架构**

- **架构类型：** Transformer（BERT变体）
- **具体模型：** CEHR-BERT（基于原始 BERT 架构改造），5层编码器，8个注意力头，隐藏维度128
- **模型参数量：** 约 **9,085,364**（可训练参数，见 Table 9）

**3.2 训练策略**

- **预训练方式：** 迁移学习基础上的领域自适应预训练（从头在 EHR 数据上预训练）
- **是否使用提示工程：** 否
- **是否涉及多阶段训练：** 是
  - 第一阶段：在 2.4M 患者数据上预训练（MLM + VTP 双目标）
  - 第二阶段：在下游任务上联合 Bi-LSTM 进行微调（fine-tuning）

**3.3 多模态信息**

- **涉及的模态：** 结构化文本（ICD诊断码、CPT操作码、药物代码）
- **融合方式：** 早期融合（概念、时间、年龄嵌入在输入层拼接后经FC层融合）
- **对齐策略：** 拼接（Concatenation）+ 全连接层投影，基于 time2vec（Fourier 分解）编码时间数值

> 本文**不涉及图像等异构模态**，属于纯结构化 EHR 的单模态时序建模。

## 五、实验与结果

**5.1 评估指标**

- **ROC-AUC**（受试者工作特征曲线下面积）
- **PR-AUC**（精确率-召回率曲线下面积，用于处理类别不平衡问题）
- 4折交叉验证（75:10:15 训练/验证/测试划分）

**5.2 对比基线**

| 类型 | 模型 |
|------|------|
| 频率特征模型 | 逻辑回归（LR）、XGBoost |
| 序列模型 | Bi-LSTM（含时序注意力预训练嵌入） |
| BERT变体 | BEHRT、MedBert、R-BERT（无预训练版CEHR-BERT） |

**5.3 核心结果**

> CEHR-BERT 在全部 4 项预测任务（T2DM→HF、HF再入院、出院后死亡、住院风险）的 AUC 和 PR-AUC 上均达到最优。尤其在 T2DM→HF 任务中，AUC 达 **80.7%**，PR-AUC 达 **32.3%**，分别超越第二名 MedBert 约 **2.5%** 和 **5%**。此外，仅用 **5% 训练数据**微调的 CEHR-BERT，其 AUC（≈0.78）仍超越使用全量数据训练的其他所有对比模型。

**5.4 消融实验**

共设计 6 个消融变体，验证了以下模块的有效性：

| 消融变体 | 移除内容 | 结论 |
|---------|---------|------|
| M-BERT | 替换为 MedBert 患者表示（无时间token） | 性能下降，ATT有效 |
| B-BERT | 替换为 BEHRT 患者表示（SEP token） | 性能低于 ATT 方案 |
| NS-BERT | 移除 VTP 第二学习目标 | 多数任务性能下降，VTP有效 |
| NT-BERT | 移除时间/年龄嵌入 | T2DM→HF PR-AUC 下降4%，时序嵌入有效 |
| ALT-BERT | 嵌入相加替代拼接+FC | 性能显著下降，FC融合优于求和 |
| V-BERT | 移除 VS/VE 访问边界 token | 性能接近 B-BERT，边界 token 不可缺少 |

## 六、局限性与不足

**作者自述：**
- 上下文窗口限制为 300 token，长病史患者存在截断问题，部分早期记录被丢弃
- 实验数据仅来自单一医疗机构（CUIMC-NYP），泛化性有待验证
- 未纳入实验室检测数据（labs），因其为连续型变量，需要不同的嵌入策略

**整理者补充：**
- 模型参数量约 900 万，规模较小，与当前大语言模型相比表达能力有限，难以处理复杂推理任务
- VTP 目标的改进幅度较小（部分任务差异在随机误差范围内），其实际贡献尚需更严格的统计检验
- 隐私保护机制未明确说明，限制了模型在多机构联合训练场景下的直接应用

## 七、挑战与未来方向

- **长序列建模：** 计划引入滑动窗口策略 + 1D卷积/LSTM 聚合区域表示，突破 512 token 的 BERT 序列上限
- **实验室数据融合：** 探索连续型数值特征（如 lab 值）的嵌入方案，丰富患者表示
- **VTP 阈值优化：** 当前 visit token 掩码率固定为 50%，需系统性探索最优阈值
- **跨机构泛化：** 基于 OMOP 通用数据模型，计划在 OHDSI 网络上进行多机构复现验证
